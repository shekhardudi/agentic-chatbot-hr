==============================================================
  AGENTIC HR — RUN COMMANDS
==============================================================

PREREQUISITES
-------------
- Docker Desktop running
- Python 3.12+

--------------------------------------------------------------
STEP 0 — Create local Python environment (project root)
--------------------------------------------------------------

  cd /path/to/agentic_hr
  python -m venv .venv
  source .venv/bin/activate
  pip install -r backend/requirements.txt -r ui/requirements.txt -r ingestion/requirements.txt pytest

--------------------------------------------------------------
STEP 1 — Create .env
--------------------------------------------------------------

  cp .env.example .env

Set at least:
  - LLM_PROVIDER=anthropic or openai
  - ANTHROPIC_API_KEY (if anthropic)
  - OPENAI_API_KEY (if openai)

Then set service tokens after first-time service setup:
  - NOCODB_API_TOKEN
  - GITEA_ADMIN_TOKEN
  - MATTERMOST_ADMIN_TOKEN

Optional but recommended:
  - NOCODB_BASE_ID

--------------------------------------------------------------
STEP 2 — Start infrastructure (Postgres, NocoDB, Gitea, Mattermost)
--------------------------------------------------------------

  docker compose up -d

Check all containers are healthy:

  docker compose ps

Services available after startup:
  NocoDB      → http://localhost:8080   (create admin account on first visit)
  Gitea       → http://localhost:3000   (run setup wizard on first visit)
  Mattermost  → http://localhost:8065   (create admin account on first visit)
  PostgreSQL  → localhost:5432

--------------------------------------------------------------
STEP 3 — After first-visit setup, generate API tokens
--------------------------------------------------------------

NocoDB:
  → Settings → Team & Auth → API Tokens → Add Token
  → Set NOCODB_API_TOKEN in .env

Gitea:
  → User Settings (top-right) → Applications → Generate Token
  → Set GITEA_ADMIN_TOKEN in .env

Mattermost:
  → System Console → Integrations → Personal Access Tokens
  → Set MATTERMOST_ADMIN_TOKEN in .env

--------------------------------------------------------------
STEP 4 — Generate synthetic data
--------------------------------------------------------------

  cd /path/to/agentic_hr
  source .venv/bin/activate
  python synthetic_data/generate.py

Output files: synthetic_data/output/*.json
SQL fallback: synthetic_data/sql/seed.sql

--------------------------------------------------------------
STEP 5 — Seed PostgreSQL with synthetic data
--------------------------------------------------------------

  docker exec -i agentic_hr_postgres psql -U agentic_hr -d agentic_hr < synthetic_data/sql/seed.sql

NocoDB is configured with PostgreSQL as its external data source,
so seeded data is visible in NocoDB automatically — no separate
NocoDB seeding step is needed.

--------------------------------------------------------------
STEP 6 — Ingest HR policy PDFs
--------------------------------------------------------------

  cd /path/to/agentic_hr
  source .venv/bin/activate
  python -m ingestion.ingest

PDFs are read from: hr_policy_pdfs/*.pdf
Parsed markdown saved to: ingestion/output_markdown/

--------------------------------------------------------------
STEP 7 — Start the FastAPI backend
--------------------------------------------------------------

  cd /path/to/agentic_hr
  source .venv/bin/activate
  cd backend
  uvicorn main:app --reload --port 8000

API docs:  http://localhost:8000/docs
Health:    http://localhost:8000/health

--------------------------------------------------------------
STEP 8 — Start the Streamlit UI
--------------------------------------------------------------

  cd /path/to/agentic_hr
  source .venv/bin/activate
  cd ui
  streamlit run app.py --server.port 8501

UI:        http://localhost:8501
  → Chat page:      home page
  → Approvals page: sidebar → Manager Approvals

--------------------------------------------------------------
STEP 9 — Run tests
--------------------------------------------------------------

  cd /path/to/agentic_hr
  source .venv/bin/activate
  pytest -q tests
